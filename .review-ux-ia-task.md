# UX/IA 错位 + 死代码 + 不兑现承诺多模型审查任务

## 触发事件

用户刚发现一个奇葩 UX bug：

**产品中心 → 我的产品** 列表底部有一个空"余额"区块，里面嵌入了 `BalanceConsumePreferences` 完整配置面板。

问题：
1. **错位**：余额不是产品（订阅/增量包才是）
2. **冗余**：顶栏已显示余额数字
3. **不兑现承诺**：subtitle 写"在账号设置中开启"，但账号设置页（AccountProfile）**根本没这功能入口**
4. **死代码风险**：`BalanceConsumePreferences` 组件全项目唯一引用就是这个错位

已修：删除错位区块、把 `BalanceConsumePreferences` 挪进 AccountProfile 兑现承诺。

## 审查诉求

**用户原话**："不要让我再看到这种奇葩的事情！"

请深度扫整个项目，找类似的：

### A. 信息架构错位（IA mismatch）
1. 组件出现在不属于它语义的页面（账户配置 vs 产品 vs 仪表盘 vs admin）
2. admin-only 功能错放进普通用户视图（或反过来）
3. 同一功能有**多个**入口（且其中有的入口已废弃/坏）
4. 同一功能"应该有但根本没有"入口

### B. 死代码 / 残留
1. 组件 / 函数 / 路由只在**一处**被引用，且该引用本身是 UX bug（说明这玩意儿可能不该存在）
2. 历史 import 没清掉（component 被删除但 import 还在）
3. i18n key 引用了不存在的翻译（或翻译引用了不存在的 component）
4. 后端 route 注册但 controller 没人调（或反过来 controller 存在但没 route）
5. SysConfig key 在 Settings.jsx 显示但后端没读取（或反过来）

### C. 文案承诺与实际功能不一致
1. 提示文案说"去 X 配置"——但 X 里没这个 UI
2. 错误消息提示用户"联系客服"——但客服系统不存在/未启用
3. 按钮文案/i18n key 暗示某功能存在，但 onClick handler 是空 / 跳转 404
4. 后端 message_code 前端没翻译（直接显示英文 code 给用户看）
5. tooltip / hint / placeholder 引用了已删除的字段名

### D. 重复 UI / 重复后端逻辑
1. 同一数据在多个组件分别 fetch（应该共享 hook / context）
2. 同一计算（如 quota 格式化）在多处重复实现，存在不一致风险
3. 多个 admin 表格 / 列表组件用同样的列定义但单独维护
4. 后端 controller 多个 endpoint 复制同样的"事务包 + 锁 user + 写 billing"模板，是否该提炼 helper

### E. 用户旅程断点
1. 注册流程中：成功后跳转的页面不存在/为空
2. 充值成功后没明确指引下一步（去用？去购套餐？）
3. 余额不足时的 402 提示是否引导到充值/订阅入口
4. 404 / 500 / 403 错误页是否友好（vs 直接 stack trace）

### F. 视觉一致性回归
1. 同一类操作（如"删除"按钮）颜色/位置/确认弹窗模式不一致
2. 同一类金额展示（USD / RMB）小数位、千分位、负数表现不一致
3. 同一类表格的分页 / 排序 / 加载态 UI 不一致
